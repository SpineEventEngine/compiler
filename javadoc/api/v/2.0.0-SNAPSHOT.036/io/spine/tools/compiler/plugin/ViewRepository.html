public class ViewRepository<I extends Object, V extends View<I, S, ?>, S extends ProjectionState<I>>
extends ProjectionRepository<I, V, S>

    A repository responsible for a certain type of Views.

    By default, when an event is headed towards a View, we try to match the first field of the event with the ID of the view. This is called routing. If the types match, the event is routed to the view with the matching ID. Otherwise, an error occurs.

    To change the rules of event routing, do one of the following:

    • Add static methods annotated with @Route in the view class.

    • Override the ViewRepository.setupEventRouting method.

    If no customization is required from a ViewRepository, users should prefer ViewRepository.default to creating custom repository types.
