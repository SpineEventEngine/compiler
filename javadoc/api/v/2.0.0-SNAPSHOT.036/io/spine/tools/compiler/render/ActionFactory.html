Package io.spine.tools.compiler.render

Class ActionFactory

  • All Implemented Interfaces:
    
public final class ActionFactory<L extends Language, D extends ProtoDeclaration>
extends Factory<RenderAction<L, D, ?>>

    Creates instances of RenderAction specified in the given actions.

    The main use case for this is to create rendering actions for a io.spine.tools.compiler.render.Renderer which obtains the instance of Actions from code generation settings.

    The classes names of which are specified in the given actions must satisfy the following criteria:

    • Be public so that this factory can create their instances reflectively.

    • Serve the language defined by the generic parameter L.

    • Serve the Protobuf declaration specified by the generic parameter D.

    In addition to the criteria above, if an action accepts a parameter, its class must have a public constructor which accepts four parameters with the following types:

    • D with an instance of Protobuf declaration to be served by the action.

    • SourceFile<L> — the file to be handled by the action.

    • Message or a derived class which corresponds to the packed parameter associated with the class name.

    • CodegenContext.

    For action classes that do not accept parameters, the public constructor must have three parameters:

    • D with an instance of Protobuf declaration to be served by the action.

    • SourceFile<L> — the file to be handled by the action.

    • CodegenContext.
