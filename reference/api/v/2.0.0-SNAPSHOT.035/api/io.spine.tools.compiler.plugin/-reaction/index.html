Reaction

abstract class Reaction<E : EventMessage> : Reaction<E> , LoadsSettings(source)

A reaction converts one event into zero to many other events.

As a rule of thumb, a reaction should read:

Whenever <something happens>, then <something else must happen>.
Content copied to clipboard

For example:

Whenever a field option is discovered, a validation rule must be added.
Content copied to clipboard

To implement the reaction, declare a method which reacts to an event with an event:

class MyReaction : Reaction<FieldOptionDiscovered>() {

    @React
    override fun whenever(@External event: FieldOptionDiscovered): Just<ValidationRuleAdded> {
        // Produce the event.
    }
}
Content copied to clipboard

Note that when reacting on Protobuf Compiler events, one should mark them as @External. See the whole list of Protobuf compiler events in spine/compiler/events.proto.

One reaction only accepts one kind of events. Declaring multiple methods with the @React annotation causes a runtime error.

The whenever method accepts a single event and produces an Iterable of events. If you need to return a single event, use Just.

To return a few events, see the descendants of Tuple.

If there can be a few alternative events, see the descendants of Either.

If one of the options does nothing at all, use io.spine.server.event.NoReaction as the event type.

Finally, if there are multiple events of the same type, use a typed list, e.g. List<SomethingHappened>.
